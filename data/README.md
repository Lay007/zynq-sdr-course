# Data Registry

This directory intentionally does not store large raw IQ captures. Instead, it stores a registry of planned, synthetic or externally hosted datasets used by the course.

## Why a registry is used

Large IQ files can quickly make the repository difficult to clone and maintain. The registry keeps the course reproducible without committing multi-megabyte or multi-gigabyte binary captures.

## Registry file

```text
data/datasets.registry.json
```

Each entry should include:

- dataset ID;
- status;
- signal type;
- hardware source;
- IQ format;
- sample rate;
- center frequency;
- expected offset;
- external URL or access note;
- metadata path;
- checksum if available;
- intended labs.

## Dataset status values

| Status | Meaning |
|---|---|
| `synthetic` | generated by repository scripts |
| `planned` | intended future real capture |
| `external` | stored outside the repository |
| `archived` | stable dataset with checksum/DOI/link |
| `private` | not publicly available |

## Adding a dataset

1. Add an entry to `datasets.registry.json`.
2. Add metadata JSON or link to metadata.
3. Add checksum if the file is stable.
4. Document analysis command.
5. Do not commit large raw capture files.
