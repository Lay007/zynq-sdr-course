# Reviewer path

This page is for a reviewer who wants to evaluate the repository in about 10 minutes.

The goal is not to read every block. The goal is to verify that the project already demonstrates an engineering route with reproducible evidence.

## 10-minute route

Open these pages in order:

1. [Course demo dashboard](demo-dashboard.md)
2. [Course status](status.md)
3. [Model → FPGA → RF → Measurement](model-to-measurement.md)
4. [Hardware checklist](hardware-checklist.md)
5. [End-to-end SDR demo roadmap](end-to-end-demo.md)
6. [Portfolio view](portfolio-view.md)

## What each page proves

| Page | What to verify |
|---|---|
| [Demo dashboard](demo-dashboard.md) | the repository already contains executable or generated artifacts |
| [Course status](status.md) | block maturity is tracked explicitly instead of implied |
| [Model → FPGA → RF → Measurement](model-to-measurement.md) | the repository has a coherent engineering chain |
| [Hardware checklist](hardware-checklist.md) | hardware work is treated as a disciplined measurement task |
| [End-to-end SDR demo roadmap](end-to-end-demo.md) | the project aims for complete model-to-measurement closure |
| [Portfolio view](portfolio-view.md) | the repository is intentionally positioned as a serious engineering artifact |

## Fast reviewer verdict

The repository is already strong if you can confirm these points:

1. Documentation and engineering route are connected.
2. At least some labs are executable or CI-backed.
3. Hardware work is described with safety and metadata discipline.
4. The project is useful both as a course and as a portfolio artifact.
