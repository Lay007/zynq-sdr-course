# Lab 6.7 — Zero-IF artifacts: DC component, mirror and tune offset

## Цель

Понять, почему реальный zero-IF SDR может показывать артефакты, которых нет в идеальной DSP-модели.

Лабораторная выполняется сначала на синтетическом IQ-сигнале. После этого тот же подход можно применить к записи с AD9363/Zynq.

## Что моделируем

| Артефакт | Простая модель |
|---|---|
| DC component | добавить постоянное комплексное смещение `I0 + jQ0` |
| IQ amplitude imbalance | умножить I и Q на разные коэффициенты |
| IQ phase imbalance | добавить небольшую фазовую ошибку между I и Q |
| Tune offset | сместить полезный тон от 0 Hz |

## Порядок работы

1. Сгенерируйте комплексный тон в базовой полосе.
2. Постройте спектр идеального сигнала.
3. Добавьте DC component и повторите спектр.
4. Добавьте IQ imbalance и найдите зеркальный компонент.
5. Сместите тон от центра спектра.
6. Сравните, что стало легче наблюдать после tune offset.
7. Запишите выводы в формате короткого инженерного отчёта.

## Ожидаемые наблюдения

- DC component проявляется около 0 Hz.
- IQ imbalance создаёт зеркальный компонент относительно центра.
- Tune offset не исправляет RF frontend, но помогает не прятать полезный сигнал под центральным артефактом.
- Без metadata невозможно честно сравнивать повторные измерения.

## Контрольные вопросы

1. Почему в zero-IF архитектуре центр спектра становится особой областью?
2. Чем DC component отличается от полезного тона?
3. Почему IQ imbalance создаёт зеркальный компонент?
4. Какие поля нужно записать в manifest перед реальной AD9363-записью?
5. Почему нельзя делать вывод о BER/EVM, не проверив gain и bandwidth настройки?

## Мини-отчёт

Минимальный отчёт должен содержать:

- частоту тона в относительных единицах или Hz;
- sample rate;
- параметры DC component;
- параметры IQ imbalance;
- два спектра: до и после tune offset;
- короткий вывод о том, какой артефакт мешает измерению сильнее всего.

## Связь с аппаратной частью

```text
manifest -> spectrum -> artifact identification -> metric -> limitation -> next action
```
