# Skill: Make IEEE-Style Engineering Figures

Use this skill when creating, regenerating or polishing figures, diagrams, screenshots and generated plots.

## Goal

Make figures readable, reproducible and useful for engineering explanation.

## Figure quality checklist

Every figure should have:

- clear title or nearby caption;
- readable axis labels with units;
- readable tick labels;
- legend outside the data or placed without overlap;
- consistent font sizes;
- light background;
- sufficient margins;
- no clipped text;
- no excessive horizontal chains;
- a clear engineering claim.

## Plot content checklist

A plot should answer at least one engineering question:

- What frequency component is present?
- What bandwidth is occupied?
- What is the EVM/BER/SNR behavior?
- What changed before and after synchronization/filtering/mixing?
- What error appears after quantization or fixed-point implementation?
- What measurement proves that the lab works?

## Diagram rules

- Prefer vertical flow for long chains.
- Keep horizontal block count small, ideally six or fewer.
- Use short labels inside blocks and explain details in text.
- Ensure arrows have visible direction and do not cross unnecessarily.
- Avoid decorative diagrams with no verification meaning.

## Reproducibility

Prefer figures generated from scripts. When adding generated images, also add or update the generator when practical.

Recommended asset locations should follow existing repository conventions, usually under `docs/assets` or `docs/images`.

## Validation

After changing figures or image paths, run or recommend:

```bash
python tools/tasks.py docs
```

If figures are generated by labs, also run:

```bash
python tools/tasks.py labs
```

## Output format

Report:

- figure purpose;
- source/generator file;
- output asset path;
- validation command;
- remaining visual risks if not checked in MkDocs.
